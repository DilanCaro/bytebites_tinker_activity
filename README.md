This project focused in understanding how to translate plain English feature request into a very simple object oriented designed with four classes: Customer , foodItem , Menu, and transaction. 

The main concept students need to understand is each class should have a clear purpose. The UML diagram should reflect only the attributes and methods reqiured by the specification. 

Students may struggle by overcomplicating the design. Adding extra things, as the AI may suggest overly complex things as it did for my bytebites_design.md. So, AI was useful for the quick draft but then also caused issues by adding not needed things.

A key lesson is that ai suggestions are just that , suggestions, and we need to check agianst original documents and requirements.
